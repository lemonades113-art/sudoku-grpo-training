# 基于GRPO强化学习的数独求解器项目

## 项目概述

开发了一个基于Group Relative Policy Optimization (GRPO)强化学习算法的9×9数独智能求解系统，通过监督微调(SFT)和强化学习两阶段训练，实现了高准确率的数独推理能力。项目采用Qwen2.5-0.5B-Instruct作为基础模型，设计了多维度奖励函数体系，并实现了分布式训练优化。

### 2. 多维度奖励函数设计

设计了5个互补的奖励函数，总分范围[-2.0, 5.0]：

#### (1) 格式正确性奖励 (Format Correctness)
- **分值**: +1.0 / -1.0
- **评估标准**: 输出是否符合9×9数独格式规范
- **实现**: 正则表达式验证 + 结构化解析

#### (2) 软格式奖励 (Soft Format)
- **分值**: 0.0 ~ +1.0 (连续值)
- **评估标准**: 部分格式正确时的渐进式奖励
- **创新点**: 避免全或无的奖励稀疏性问题

#### (3) 数独有效性奖励 (Sudoku Validity)
- **分值**: +1.0 / -1.0
- **评估标准**: 行、列、宫内数字1-9不重复
- **实现**: 高效的约束检查算法

#### (4) 线索保留奖励 (Clue Preservation)
- **分值**: 0.0 ~ +1.0
- **评估标准**: 保留原题所有已知数字的比例
- **公式**: preserved_clues / total_clues

#### (5) 正确性奖励 (Correctness)
- **分值**: +1.0 / 0.0
- **评估标准**: 与标准答案完全一致
- **权重**: 最高优先级

**奖励函数可视化**: 生成了5张奖励曲线图，存储于`reward_images/`目录

## 技术难点与解决方案

### 难点1: 长序列生成稳定性
**问题**: 9×9数独需要生成81个数字，容易出现格式崩溃

**解决方案**:
- 设计了软格式奖励，渐进式引导
- 使用格式锚点(sudoku_format_anchor.json)作为参考
- KL散度约束防止策略偏离过远

### 难点2: 奖励稀疏性
**问题**: 完全正确才有奖励，导致训练困难

**解决方案**:
- 分解为5个子奖励函数
- 引入中间奖励(线索保留、部分正确)
- 使用组相对优势估计，降低方差

### 难点3: 显存限制
**问题**: 单卡显存不足以支持大批次训练

**解决方案**:
- DeepSpeed ZeRO-2优化器状态分片
- 梯度累积模拟大批次
- 梯度检查点降低激活值显存

### 难点4: 训练效率
**问题**: GRPO需要多次采样，训练慢

**解决方案**:
- 双卡并行训练，加速1.7倍
- Unsloth加速LoRA计算
- 优化数据加载与预处理
